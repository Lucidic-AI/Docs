## Overview

The Lucidic TypeScript SDK provides a comprehensive set of functions to track, monitor, and analyze AI agent workflows. These functions handle session management, event tracking, and integration with various LLM providers.

## Installation

```bash
npm install lucidicai
```

## Quick Start

```typescript
import { LucidicAI } from 'lucidicai';
import OpenAI from 'openai';

// Initialize client with provider
const client = new LucidicAI({ apiKey: '...', providers: ['openai'] });

// Use context manager for session lifecycle
await client.sessions.create({ sessionName: 'My AI Agent' }, async () => {
    const openai = new OpenAI();

    // LLM calls are automatically tracked as events
    const response = await openai.chat.completions.create({
        model: 'gpt-4',
        messages: [{ role: 'user', content: 'Hello!' }]
    });

    // Manual events for custom operations
    client.events.emit({
        name: 'Custom operation',
        input: { key: 'value' },
        output: { result: 'success' }
    });
});
// Session auto-ends when callback completes
```

## Function Categories

### Session Lifecycle

1. **Initialization**: Create a `LucidicAI` client and use `client.sessions.create()` to start sessions
2. **Continuation**: To resume an existing session, pass `sessionId` to `sessions.create()`
3. **Updates**: Modify session data with `client.sessions.update()`
4. **Completion**: Sessions auto-end with context managers, or call `client.sessions.end()` explicitly

### Event Tracking

1. **Automatic**: LLM calls through instrumented providers create events automatically
2. **Manual Sync**: Use `client.events.create()` for custom events
3. **Zero-Latency**: Use `client.events.emit()` for hot paths

### Utilities

1. **Prompts**: Centralized prompt management with `client.prompts.get()`
2. **Context**: Async-safe session scoping via context managers

## Provider Support

The SDK instruments providers when specified in the `providers` array:

- **OpenAI** - `providers: ['openai']`
- **Anthropic** - `providers: ['anthropic']`
- **Vercel AI SDK** - `providers: ['vercel-ai']`

For ESM/Next.js, you can also pass module references via `instrumentModules`:

```typescript
import OpenAI from 'openai';
import { Anthropic } from '@anthropic-ai/sdk';

const client = new LucidicAI({
    apiKey: '...',
    instrumentModules: { OpenAI, anthropic: Anthropic }
});
```

## Best Practices

1. **Prefer Context Managers**: Use `client.sessions.create()` with a callback for automatic cleanup
2. **Use `.emit()` for Hot Paths**: Zero-latency tracking for high-frequency events
3. **Always Initialize First**: Create the client before any SDK operations
4. **Handle Promises**: Always await async operations
5. **Type Safety**: Leverage TypeScript's type system for better development experience

## Error Handling

```typescript
import { LucidicAI, APIError } from 'lucidicai';

try {
    const client = new LucidicAI({ apiKey: '...' });

    await client.sessions.create({ sessionName: 'My Session' }, async () => {
        // Your workflow
    });
} catch (error) {
    if (error instanceof APIError) {
        console.error('API error:', error.message);
    }
}
```

## Environment Variables

- `LUCIDIC_API_KEY` - Your API key for authentication
- `LUCIDIC_AGENT_ID` - Your agent identifier
- `LUCIDIC_PRODUCTION_MONITORING` - Enable production mode
