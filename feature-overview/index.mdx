# Feature Overview

Welcome to Lucidic AI, the complete platform for AI agent observability and optimization. Explore our features to understand how Lucidic can help you build, debug, and improve your AI agents.

---

## Core Concepts

<CardGroup cols={2}>
  <Card title="Sessions" icon="play" href="/feature-overview/sessions">
    A single run of your AI agent, capturing the full execution trace from start to finish.
  </Card>
  <Card title="Events" icon="bolt" href="/feature-overview/events">
    The core tracking unit representing LLM calls, function invocations, errors, and custom operations.
  </Card>
</CardGroup>

---

## Features

<CardGroup cols={2}>
  <Card title="Workflow Sandbox" icon="microscope" href="/feature-overview/workflow-sandbox">
    Deep-dive into individual sessions to replay, inspect, and debug your agent's behavior.
  </Card>
  <Card title="Experiments" icon="flask" href="/feature-overview/experiments">
    Group sessions for bulk analysis with powerful analytics and failure detection.
  </Card>
  <Card title="Evaluation Rubrics" icon="clipboard-check" href="/feature-overview/rubrics">
    Build structured evaluation criteria to score or validate agent behavior.
  </Card>
  <Card title="Datasets" icon="database" href="/feature-overview/datasets">
    Create and manage test datasets for systematic agent validation.
  </Card>
  <Card title="Evolutionary Simulations" icon="dna" href="/feature-overview/evolutionary-simulations">
    Automatically optimize your agents through intelligent hyperparameter evolution.
  </Card>
  <Card title="Prompt Database" icon="message" href="/feature-overview/prompt-db">
    Manage, version, and fetch prompts dynamically without redeploying code.
  </Card>
  <Card title="Time Travel" icon="clock-rotate-left" href="/feature-overview/time-travel">
    Freeze time and replay any moment in your agent's execution for debugging.
  </Card>
  <Card title="Production Monitoring" icon="chart-line" href="/feature-overview/production-monitoring">
    Track performance, costs, and errors in real-time production environments.
  </Card>
</CardGroup>
